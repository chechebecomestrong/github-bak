git佛系命令

发表于 2018-08-06
字数统计: 449 | 阅读时长 ≈ 2

2018年8月6日
Git一般的使用场景：
一般是旧项目
一般直接git clone
修改完后再次上传的时候 配置文件也是之前写好的

然鹅……
今天下午，面对一个新的项目，什么都没有……然后，git就变得很佛性

分享一个佛性的我面对佛性的git的心路历程：
按部就班，
git init
git add .
git commit
然后之后git push就可以了
但是，失败了。
发现，是因为没有master。
没有master，什么都是0。
添加了master后，打了个分支，然后高高兴兴上传。
依旧失败。
查找原因，可能是本地分支没有关联远程分支的原因。
删除分支，重新创建分支。
上传成功！……但是发现：文件结构多了一层。
删除分支，重新上传。
上传成功！……但是发现：有node_modules和mac自带的文件
于是想起来要有一个gitignore文件。
写了个最简单的gitignore文件：
node_modules/
node_modules/*
.DS_Store
删掉分支，再次上传。
gitignore并没有发挥作用。
重写了gitignore但是没有用。
然后，考虑到git的缓存的问题。
于是
git status
真的是有缓存文件。
git clean出场
删除 untracked files
git clean -f

连 untracked 的目录也一起删掉
git clean -fd

连 gitignore 的untrack 文件/目录也一起删掉 （慎用，一般这个是用来删掉编译出来的 .o之类的文件用的）
git clean -xfd
以上这三条，没有起作用。
最后尝试，
git rm -r –cached .
这才终于把缓存清掉。
上传。成功！gitignore也发挥了作用。
git虽然几乎天天用，但是经常用于旧项目，在应对新项目时竟然出了这么多的幺蛾子、
基础是进步的基石，共勉！
