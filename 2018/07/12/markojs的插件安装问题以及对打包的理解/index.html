1、所有的marko都有的问题
即lasso的问题，在npm的建议下安装了最新版后，
还需要再安装一下@lasso/marko-taglib
npm install --save @lasso/marko-taglib

2、npm install的时候报错
ChromeDriver installation failed Error with http(s) request: Error: read ETIMEDOUT
解决方法为：
npm install chromedriver --chromedriver_cdnurl=http://npm.taobao.org/mirrors/chromedriver

示例项目中的打包情况如下：
demo文件夹里的browser.json文件里陈述了对应的依赖关系里有demo1.js、demo2.js和demo3.js，会有一个以demo开头的js文件。
在server.js里对demo1.js和demo2.js进行了特殊的设置，把这两个文件打包成一个以base开头的js文件。
在浏览器中查看着两个js文件中的内容，两个文件中的内容并没有相同的部门。
由此可以这么理解，在json文件中，只表明了依赖关系，如果不进行特殊设置的话，会把所有的依赖都打在同一个js文件里，但是如果在server.js进行了特殊的指定，那么被特殊指定的js文件会单独打包。
所有的依赖关系，都会被打包，并且只打包一次。
