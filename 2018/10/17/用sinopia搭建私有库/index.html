用sinopia搭建私有库

发表于 2018-10-17 | | 阅读次数:
字数统计: 792 | 阅读时长 ≈ 3

使用sinopia的优势：
配置简单
不需要数据库，sinopia内置有一个数据库了
开启即用
当源为sinopia时，本地不存在要安装的包时，会自动去npm把包缓存到本地
1、服务器安装sinopia
sudo npm install sinopia -g
如上图所示即为安装完成
运行 sinopia
这个时候会出现两个warn
一个是config
一个是地址，是本地的地址，只能本地访问。
本地访问这个接口，可以用ping的方式，显示是通的。
2、服务器配置sinopia

3、改变本地的npm源
推荐使用nrm来管理
（1）安装nrm
npm install nrm -g
(2)只需要输入 nrm
可以看到很多的命令提示
这个sinopia是安装在某个环境
所以在本地增加安装在某个的源：
nrm add sinopia http://你的地址:4873
nrm use sinopia
以上设置就可以用我们搭建好的私有npm了
安装完成之后，你通过npm install 安装的包，sinopia都会帮你缓存到本地。只要安装过，再次下载到本地会很快。
4、注册账号
确认自己的npm源在我们私有的库里。
npm adduser
按照提示，输入相关信息
这样就可以成为开发者了。
在注册的时候，很有可能注册不成功。可以用下面的方法来增加开发者。
用户名+密码->sha1
将上面的字符串写入和config.yml同目录下的htpasswd中，至此就可以增加用户了。
如果已经有账号，直接运行 npm login
5、上传自己的包
npm init
填写包信息
写完后，
运行 npm publish 发布新包
6、登录http://你的地址:4873
login→填写自己的用户名和密码，登录后就可以看当前的私有npm管理器中局域网内提交的包
7、install到本地
npm init
生成一个pakage.json
npm install
这样可以获取sinopia里所有的包的信息
如果要安装一个名字叫aa的包
那么可以运行下面的语句：
npm install aa
本地会生成一个node_module的文件夹，然后文件夹里会有aa的包
在项目中，需要用的这个包的时候，引用即可
