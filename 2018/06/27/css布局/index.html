基础是进步的基石。

看到群里有人在问有没有人有仿过百度的一整套的页面布局，结果是没有。百度作为日常用的最多的网站之一，在群里没有一个人回答，可能仿百度的人应该也少，毕竟没有谁在用不着那样的布局去仿一个工作上用不到的网站。我以为是类似于作业一类的东西，毕竟作业那种东西，随便来点就行了，了解过后，竟然真的有个公司的产品设计图给前端的就是类似百度的排版布局。
说到仿大网站的排版布局，在大四的时候因为涉及到毕业设计的排版，我曾经仿过京东的页面布局，因为京东各种改版，最新版的京东的页面也和我仿的那一版有很多出入了。突然就想到自己最近都没怎么仿过大网站的布局和排版，于是有了仿个百度试试的想法。然后，就没有了然后，随着前端框架的流行，我现在仿排版布局的能力竟然比不上大四的我。再多的框架的流行，出现再多的新技术，都不能忘记基础。
痛定思痛，从菜鸟迈向小白之路，基础是最不能丢的一部分。
所以，就先从基础开始记录吧。

从2018年6月开始挖坑，一直到2019年10月终于想起来填坑，很多事情，放着放着就放没了，但是，更多的事情，放着放着还得做。

作为2017年毕业的学生，我是听着“组件化”这个名词走过来的，组件化这个词越来越多的被提及，js就被越来越多的重视，很多人在重视js的时候，我看到了css的强大。对，我想说的是，css越来越强大了，强大到他的变化你感知不到，如果要实现一个什么效果用js很困难的时候，css的一个属性就可以解决。

废话不多说，正文开始：

display

display是CSS中最重要的用于控制布局的属性，每个元素都有一个默认的display值，这与元素的类型有关，大多数元素它们的默认值通常是block或者inline。一个block元素通常被叫做块级元素，一个inline元素通常被叫做行内元素。
block
div是一个标准的块级元素。一个块级元素会新开始一行并且尽可能撑满容器。其他常用的块级元素包括p、form和HTML5中的新元素：header、footer、section等等
inline
span是一个标准的行内元素。一个行内元素可以在段落中zz包裹一些文字而不会被打乱段落的布局。a元素是最常用的行内元素，它可以被用作链接。
none
另一个常用的display值是none。一些特殊元素的默认display值就是它。display：none通常被JavaScript用来在不删除元素的情况下隐藏活显示元素。它和visibility属性不一样。把display设置成none元素不会占据它本来应该显示的空间，但是设置成visibility：hidden，还会占据空间。
其他display值

其他
每个元素都有一个默认的display类型，不过可以随时随地的重写。

margin：auto

设置块级预算元素的width可以防止它从左到右撑满整个容器。然后你就可以设置左右外边距为auto来使其水平居中。元素会占据你所指定的宽度，然后剩余的宽度会一分为二成为左右外边距。
唯一的问题是，当浏览器窗口比元素的宽度还要宅时，浏览器会显示一个水平滚动条来容纳页面。然后就有了以下的解决方案。

max-width

在这种情况下使用max-width替代width可以使浏览器更好的处理小窗口的情况，在移动设备上显得尤为重要。
所有的主流浏览器包括IE7+在内都支持max-width，所以可以放心使用

盒模型

当你设置了元素的宽度，实际展现的元素却超出了你的设置：这是因为元素的边框和内边距会撑开元素。
在以前，开发者需要用比他们实际想要的宽度小一点的宽度，需要计算相应的宽度，即减去内边距和边框的宽度，但是现在不需要再这么做了

box-sizing

人们慢慢地意识到传统的和子模型不直接，所以新增了一个box-sizing的css属性。当你设置一个元素为box-sizing: border-box;时，此元素的内边距和边框不会再增加它的宽度。

position

  1. static
    static是默认值，任意的position：static的元素不会被特殊的定位。一个static元素表示它不会被“positioned”，一个position属性被设置为其他值的元素表示它会被“positioned”。
  2. relative
    relative表现的和static一样，除非你添加了一些额外的属性。在一个相对定位的元素上设置top、bottom、left、right属性会使其偏离其正常位置。其他的元素的位置则不会受该元素的影响发生位置改变来弥补它偏离后剩下的空隙。
  3. fixed
    一个固定定位的元素会相对于视窗来定位，这意味着即便页面滚动，它还是会停留在相同的位置，和relative一样，top、bottom、left、right属性都可用。
    一个固定定位的元素不会保留它原本在页面中应有的空隙，即脱离文档流。
    但是移动浏览器对fixed的支持很差，尤其是前期ios系统对webview进行的“优化”。对于移动端页面开发，这个属性需要对各种手机型号、各种手机系统以及客户使用的版本进行测试。必要的时候还是使用position：absolute，让这个元素回归文本流。
  4. absolute
    absolute是最棘手的position值。与fixed的表现类型，但是它不是相对于视窗而是相对于最近的“positioned”祖先元素。如果绝对定位的元素没有“positioned”祖先元素，那么它是相对于文档的body元素并且它会随着页面滚动而滚动。

媒体查询

响应式布局是一种让网站针对不同的浏览器和设备呈现不同显示效果的策略，这样可以让网站在任何情况下显示的符合预期。
