parcel：一款又快又简单的webapp打包工具

发表于 2018-09-07
字数统计: 507 | 阅读时长 ≈ 2

https://medium.freecodecamp.org/all-you-need-to-know-about-parcel-dbe151b70082

可以看一下这篇文章的写作时间：2017年12月
距离现在整整大半年了。
国内还没有流行起来。但是看了作者的介绍，就感觉这个东西，还不错。
对这个打包工具
作者用了这样的形容词——blazing fast、zero configuration
—————————-我是萌萌哒分割线，下面就开始说正事了———————————————
一说到打包，第一反应webpack，然后是browserify。
然后现在，又有一个工具进入了我们的视线。——parcel
webpack以复杂的配置和文件大著名。
browserify以入门的门槛低而著名。
作者用一个有1726个模块在16年款的MacBook pro上运行打包，结果显而易见
bundler time
browserify 22.98s
webpack 20.71s
parcel 9.98s
parcel-with cache 2.64s
在速度方面，parcel表现的很棒。
除了速度，parcel不仅仅支持js，css，html，配置等文件
对使用者更加友好：不用额外的插件
因为没有插件，所以也不需要配置。
支持热重启，css预编译、服务器、缓存等……
这么友好的工具，在错误提示上也更加友好
工具是否好用是离不开具体的环境的。
作者给出了这样的建议：
如果代码行数小于15k，建议用parcel
如果项目是大的企业级的，建议用webpack
如果对于npm包，建议用rollup。
下面是quick start：
1、安装
npm Install parcel-bundler –save-dev
2、在打包的时候要注意，要在inde.html中引用index.js，这样算是唯一的需要配置的地方了
3、在package.json里面可以在脚本的start后面写上这样一种操作
“start”:{“start”:”parcel index.html”}
运行后会出现一个dist文件夹，包含一个HTML文件一个js文件和一个css文件。
而且也支持sass、react、vue、typescript等文件的打包。
重大发现：原来原来原来我们很久之前的一个没有解决的问题，可以被解决了！
