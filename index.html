二、ts手册

基本类型

在编程中，需要使用一些最简单的数据单元：数字、字符串、结构、布尔值等等。在ts中，有和js差不多的基本类型，和更方便的枚举类型。
布尔值
最基础的数据类型，只有简单的两个值：true和false。在js和ts中，都叫做布尔值。
let isDone: boolean = false;
数值
js中和ts中的数都是浮点数。这些浮点数的类型叫“number”，除了十六进制和十进制，ts还支持es2015引入的二进制和八进制。

字符串
字符串是不论由js写成的网页还是服务器端运行的程序的另一个基本部分。和其他的语言一样，我们用string来表示字符串。和js一样，ts也是用单引号或者双引号来包住字符串。

当然还可以使用模版字符串，可以跨越多行并具有嵌入式表达式这些字符串用反引号包住，并且嵌入式的表达形式为${expr}

数组
ts和js一样，允许程序员用数组来编写。数组类型可以用一下两种方式来编写：

  1. 使用的元素的类型后跟[]表示该元素类型的数组
    let list: number[] = [1, 2, 3];
  2. 使用通用数组类型Array<elemType>
    let list: Array<number> = [1, 2, 3];

元组
元组类型允许开发者用有限的已知类型的元素来表示数组，类型不用一致。例如可以用一个字符串和一个数字来表示一个值。

任意值（Any）
在编写程序的时候我们需要去描述一些我们不知道的类型。这些值可能是变化的内容，来自用户或者第三方库。在这些情况下，希望不去检查值得类型，并且让这些值通过编译检查，这个时候需要用any来标记这些值。

任意值类型是一个很有效的方法，尤其是用现有的js的开发者，可以逐步的在编译的时候选择加入或者去掉类型检查。你可能期望用对象来扮演一个差不多的角色，在其他的语言里的确可以做到这一点。然而，对象类型只允许复制操作，不能调用任意方法，即使是已经存在的方法。

注意：要避免使用Object类型，非原始的对象类型。

Null和Undefined
在ts里，undefined和null通常有他们的类型名：undefined和null。很像void，单纯用他们本身，是没用的，一般用于对返回值的处理。

一般情况下，null和undefined是其他类型的子类型。这意味着可以给其他的类型比如数值类型的变量赋null或者undefined。
然而，当用–strictNullChecks这个标志的时候，null和undefined就只能赋值给any和各自的类型（除了undefined也可以分配给void）。这有助于避免许多常见的错误，如果要传递一个值或者是字符串、或者是null或者是undefined的时候，可以用联合类型，string|null|undefined。
联合类型是一个进阶的话题，这里不继续学习。
注意：ts官方是鼓励可能的话，使用--strictNullChecks标志。

JSDoc是一个根据javascript文件中注释信息，生成JavaScript应用程序或库、模块的API文档 的工具。你可以使用他记录如：命名空间，类，方法，方法参数等。类似JavaDoc和PHPDoc。现在很多编辑器或IDE中还可以通过JSDoc直接或使用插件生成智能提示。从而使开发者很容易了解整个类和其中的属性和方法，并且快速知道如何使用，从而提高开发效率，降低维护成本。

http://momentjs.com/
moment.js不依赖任何第三方库，支持字符串、Date、时间戳以及数组等格式，可以像PHP的date()函数一样，格式化日期时间，计算相对时间，获取特定时间后的日期时间等等，本文有如下举例。
格式化日期
当前时间：
moment().format('YYYY-MM-DD HH:mm:ss');
今天是星期几：
moment().format('d');
转换当前时间的Unix时间戳：
moment().format('X');
相对时间
20120907相对当前日期是6年前
moment("20120907", "YYYYMMDD").fromNow(); //6 years ago
7天后的日期：
moment().add('days',7).format('YYYY年MM月DD日');
9小时后的时间：
moment().add('hours',9).format('HH:mm:ss');
moment.js提供了丰富的说明文档，使用它还可以创建日历项目等复杂的日期时间应用。

对这个打包工具
作者用了这样的形容词——blazing fast、zero configuration

下面开始说正事：
一说到打包，第一反应webpack，然后是browserify。
然后现在，又有一个工具进入了我们的视线。——parcel
webpack以复杂的配置和文件大著名。
browserify以入门的门槛低而著名。
作者用一个有1726个模块在16年款的MacBook pro上运行打包，结果显而易见
bundler time
browserify 22.98s
webpack 20.71s
parcel 9.98s
parcel-with cache 2.64s
在速度方面，parcel表现的很棒。
除了速度，parcel不仅仅支持js，css，html，配置等文件
对使用者更加友好：不用额外的插件
因为没有插件，所以也不需要配置。
支持热重启，css预编译、服务器、缓存等……
这么友好的工具，在错误提示上也更加友好
工具是否好用是离不开具体的环境的。
作者给出了这样的建议：
如果代码行数小于15k，建议用parcel
如果项目是大的企业级的，建议用webpack
如果对于npm包，建议用rollup。
下面是quick start：
1、安装
npm Install parcel-bundler –save-dev
2、在打包的时候要注意，要在inde.html中引用index.js，这样算是唯一的需要配置的地方了
3、在package.json里面可以在脚本的start后面写上这样一种操作
“start”:{“start”:”parcel index.html”}
运行后会出现一个dist文件夹，包含一个HTML文件一个js文件和一个css文件。
而且也支持sass、react、vue、typescript等文件的打包。
