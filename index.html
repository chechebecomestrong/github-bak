js模版引擎——artTemplate

发表于 2018-11-12
字数统计: 416 | 阅读时长 ≈ 2

简介
artTemplate是一款简约、超快的模版引擎。采用作用于预声明的技术来优化末班渲染速度，从而获得接近JavaScript极限的运行性能，并且同时支持nodejs和浏览器。
调试时的错误日志精确到模版所在行，调试友好。支持在模版文件上打断点。
支持模版集成与子模版。
浏览器版本，仅6KB。
模版：
支持两种模版语法：标砖语法和原始语法。
原始语法大多用于EJS模版。
核心方法：
//基于模版名渲染模版
template(filename,data);
//将模版源代码编译成函数
template.compile(source,options);
//将模版源代码编译成函数并立刻执行
template.render(source,data,options);
安装：
NPM:
npm install art-template –save
浏览器：
template-web.js
差异：
因为浏览器不支持文件系统，所以template(filename,data）不支持传入文件路径，它内部使用document.getElementById(filename).innerHTML来获取模版。
可以这样写：
<!DOCTYPE html>




art-template写法示例

<body>
<div class="app"></div>
<script id="test" type="text/html">
<h1>{{#title}}</h1>
</script>
<script>
var data = {
title: "<p>hello world</p>"
};
var html = template("test", data);
document.getElementById('app').innerHTML = html;
</script>
</body>

运行结果就是在网页中渲染出了
hello world
*需要注意的是：如果在模版中使用js模版引擎——artTemplate，会渲染出

hello world


所以值得注意的是，数据需不需要原文输出。
也可以在浏览器中预编译，这种预编译是使用了webpack的loader
http://aui.github.io/art-template/zh-cn/docs/index.html
更多详细的可以看一下官方的文档。
对于模版，我们可以做的还有很多。

运行 sinopia
这个时候会出现两个warn

本地访问这个接口，可以用ping的方式，显示是通的。

3、改变本地的npm源
推荐使用nrm来管理
（1）安装nrm
npm install nrm -g
(2)只需要输入 nrm
可以看到很多的命令提示
这个sinopia是安装在某个环境
所以在本地增加安装在某个的源：
nrm add sinopia http://你的地址:4873
nrm use sinopia
以上设置就可以用我们搭建好的私有npm了
安装完成之后，你通过npm install 安装的包，sinopia都会帮你缓存到本地。只要安装过，再次下载到本地会很快。
4、注册账号
确认自己的npm源在我们私有的库里。
npm adduser
按照提示，输入相关信息
这样就可以成为开发者了。
在注册的时候，很有可能注册不成功。可以用下面的方法来增加开发者。
用户名+密码->sha1
将上面的字符串写入和config.yml同目录下的htpasswd中，至此就可以增加用户了。
如果已经有账号，直接运行 npm login
5、上传自己的包
npm init
填写包信息
写完后，
运行 npm publish 发布新包
6、登录http://你的地址:4873
login→填写自己的用户名和密码，登录后就可以看当前的私有npm管理器中局域网内提交的包
7、install到本地
npm init
生成一个pakage.json
npm install
这样可以获取sinopia里所有的包的信息
如果要安装一个名字叫aa的包
那么可以运行下面的语句：
npm install aa
本地会生成一个node_module的文件夹，然后文件夹里会有aa的包
在项目中，需要用的这个包的时候，引用即可

JSDoc——自动化文档工具

发表于 2018-09-13
字数统计: 746 | 阅读时长 ≈ 3

不管是写组件还是写接口，最后都要回归到文档。
我们对于文档的需求：
1、最好是自动生成的

2、生成的最好是html的

这类需求，催生了一类工具的产生，这类工具的名字为——自动化文档工具

这类工具中，推荐一个相对来说适合前端的——JSDoc
JSDoc是一个根据javascript文件中注释信息，生成JavaScript应用程序或库、模块的API文档 的工具。你可以使用他记录如：命名空间，类，方法，方法参数等。类似JavaDoc和PHPDoc。现在很多编辑器或IDE中还可以通过JSDoc直接或使用插件生成智能提示。从而使开发者很容易了解整个类和其中的属性和方法，并且快速知道如何使用，从而提高开发效率，降低维护成本。

在node中使用
1、安装
npm install -g jsdoc
或者
npm install –save-dev jsdoc
2、编写函数说明
按照jsdoc规定的语法进行编写
http://www.css88.com/doc/jsdoc/about-plugins.html
所有的函数说明都写在这样的注释里/ */
举个栗子：
/ 这是一个测试的函数 /
function test(){}
再举一个栗子：
/** 这是一个函数

  • @param a {string} a的解释
  • @returns {string} 对于返回值的解释
    */
    function test(a){
    return a+’hello’;
    }
    常用的语法：
    @param 记录传递给一个函数的参数
    @returns 记录一个函数的返回值
    @requires 这个文件需要一个JavaScript模块
    @function 描述一个函数或方法
    @exports 标识一个由JavaScript模块到处的成员
    @enum 描述一个相关属性的集合
    @example 提供一个如何使用描述项的例子
    @class 此函数旨在需要使用new关键字调用，即构造函数，也可以写@constructor
    @listen 列出一个标识的监听事件
    @property 记录一个对象的属性
    @see 更多详细信息请参阅其他一些文档
    @global 记录一个全局对象

3、生成

jsdoc 文件(路径)

使用jsdoc模版：
默认的模版不够美观，所以为了更好看，可以使用其他的模版：
可以去Github jsdoc3的readme.md中寻找其他可用模版。
比如：喜欢的模版名称为 dd
可以这样安装：
npm install dd
然后在根目录下新建一个jsdoc.config.json
这个json可以参照官方用模版时的json的设置：
主要包括
source：jsdoc的源
opts：选项设置
templates：和模版相关的可选项的设置
模版中也可以用插件，最常见的应该是markdown
因为用markdown里的readme的内容可以插入到html中作为首页。让文档内容显示更加直观。
配置好jsdoc.config.json后，
可以用下面的语句来生成html文档：
jsdoc -c jsdoc.config.json
到此为止，文档的框架就美美的生成了。
以后的优化就在于对README.md的优化和对注释的优化了。

http://momentjs.com/
moment.js不依赖任何第三方库，支持字符串、Date、时间戳以及数组等格式，可以像PHP的date()函数一样，格式化日期时间，计算相对时间，获取特定时间后的日期时间等等，本文有如下举例。
格式化日期
当前时间：
moment().format(‘YYYY-MM-DD HH:mm:ss’);
今天是星期几：
moment().format(‘d’);
转换当前时间的Unix时间戳：
moment().format(‘X’);
相对时间
20120907相对当前日期是6年前
moment(“20120907”, “YYYYMMDD”).fromNow(); //6 years ago
7天后的日期：
moment().add(‘days’,7).format(‘YYYY年MM月DD日’);
9小时后的时间：
moment().add(‘hours’,9).format(‘HH:mm:ss’);
moment.js提供了丰富的说明文档，使用它还可以创建日历项目等复杂的日期时间应用。

bundler time

工具是否好用是离不开具体的环境的。
作者给出了这样的建议：
如果代码行数小于15k，建议用parcel
如果项目是大的企业级的，建议用webpack
如果对于npm包，建议用rollup。
下面是quick start：
1、安装
npm Install parcel-bundler –save-dev
2、在打包的时候要注意，要在inde.html中引用index.js，这样算是唯一的需要配置的地方了
3、在package.json里面可以在脚本的start后面写上这样一种操作
“start”:{“start”:”parcel index.html”}
运行后会出现一个dist文件夹，包含一个HTML文件一个js文件和一个css文件。
而且也支持sass、react、vue、typescript等文件的打包。

ui自动化测试探索——ui recorder

发表于 2018-09-04
字数统计: 1,209 | 阅读时长 ≈ 4

最近在写npm包相关的内容。
在写到单元测试相关内容的时候，想到当前的单元测试只限于检查一系列的js代码的正确性和相关性。
于是，想了下能不能对UI进行自动化测试？
答案是肯定的。
首当其冲的，则是UI Recorder。
是阿里巴巴的开源项目。
官方的介绍十分的高大上：
UI Recorder 是一款零成本UI自动化录制工具，类似于Selenium IDE.
UI Recorder 要比Selenium IDE更加强大!
UI Recorder 非常简单易用.
官方网站: http://uirecorder.com/
语言切换: English, 中文
变更日志: CHANGE
视频教程：PC中文教程
功能
支持所有用户行为: 键盘事件, 鼠标事件, alert, 文件上传, 拖放, svg, shadow dom
支持无线native app(Android, iOS)录制, 基于macaca实现: https://macacajs.com/
无干扰录制: 和正常测试无任何区别，无需任何交互
录制用例存储在本地
支持丰富的断言类型: val,text,displayed,enabled,selected,attr,css,url,title,cookie,localStorage,sessionStorage
支持图片对比
支持强大的变量字符串
支持公共测试用例: 允许用例中动态调用另外一个
支持并发测试
支持多国语言: 英文, 简体中文, 繁体中文
支持单步截图
支持HTML报告和JUnit报告
全系统支持: Windows, Mac, Linux
基于Nodejs的测试用例: jWebDriver
因为node操作起来相对较为简便。所以选择用node来探索。
工欲善其事必先利其器
1、安装
首先安装node 安装Chrome 安装ui recorder、mocha
安装完之后，需要对环境进行补充，需要另外安装Java sdk，jwedriver
在把环境都安装好之后，就可以进行测试了。
2、测试步骤
首先，新建一个文件夹。
然后进入测试环节：
（1）uirdcorder init 配置好之后在用ui recorder的环境 会生成一些配置文件
（2）生成的json文件中，可以修改config.json使配置更加符合自己的情况，而且也可以修改在init的时候配置错误的选项。
（3）修改好配置文件后，就可以开始录制测试用例了。
指令为 uirecorder start 测试用例名
测试用例名以.spec.js结尾
然后会有一些关于录制时的配置需要进一步的确定。都确认之后，终端会成为所谓的录制服务器，内部的驱动会拉起系统的浏览器。
这个时候会拉起两个浏览器，一个是录制浏览器，一个是检验浏览器，所有的录制动作都是在录制浏览器进行的，检验浏览器会重复录制浏览器的动作，然后在录制服务器中显示测试结果。
录制浏览器会有一个输入框，即输入要测试的内容，这里可以是一个url，也可以是一个脚本。点击开始录制即真正了进入了测试过程。
录制界面会有一排按钮框，悬浮可移动。
有添加悬停、添加断言、使用变量、脚本跳转、结束录制等功能。
在经过一些列的测试之后，可以结束录制。这个时候在录制中的所有的操作都以代码的形式出现在刚刚的.spec.js文件中。
所以，这个文件可以用来回看。回看需要mocha。
mocha 测试用例名
这样就可以回看刚刚的录制。
然后在终端，会出现各种测试动作的结果。
有mocha的加入，就有了可以文本化测试结果的可能。
source run.sh 测试用例名
这个时候除了会回放录制的动作，回放完后会生成一份测试报告
测试报告中除了一般的单元测试测试报告中显示的各项数据外，还有录制动作的截图。
可以实时、带代码块、带测试结果的了解整个测试过程。

软件也都不是完美的：
这款测试框架：
只适用于一般的dom操作，以及一般的dom写的ui。
不适合我们公司的ui框架。因为我们公司的ui框架写的很精细，正常的写法测不出来。
如果硬写也能写出来，但是到时候测试的代码可能比开发代码还多。这样就会导致测试的成本过高。
因此，这个ui测试，以后可以在表格、一般的dom操作中使用。不适合用来测试本公司前端的ui框架。
所以，这个ui测试框架，不再做进一不的探究。需要的时候，用其基本的写法即可。
